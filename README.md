# Mermaid Studio

> 一个离线单文件的 Mermaid 图表工坊。双击 HTML 就能用。

把 Mermaid 代码粘进左边，图就在右边。完全离线、不依赖 CDN、不需要服务器，整个工具就是一个 3.3 MB 的 HTML 文件。

## 使用

### 只想用 / 看图（绝大多数同事）

你只需要 **`mermaid-studio.html`** 这一个文件。

1. 拿到 `mermaid-studio.html`（同事发你、SVN/Git checkout、或网盘下载都行）
2. 放到本机任意位置——**放哪都行，不依赖任何固定路径或其它文件**
3. 双击，用浏览器（Chrome / Edge 等）打开
4. 左边粘 Mermaid 代码、或点顶部"**载入封神图表**"选一张，右边即时出图
5. 想偷懒？点顶部 **「✦ AI 助手」**，用一句话描述你要的图，AI 自动生成代码并渲染（需先配置自己的大模型 API，见下）

无需联网、无需安装、无需服务器。这个 HTML 是自包含的（约 3.3 MB，已内置 mermaid 库和所有业务图表）。

> 想把工具发给别人？直接把 `mermaid-studio.html` 单个文件发过去即可，对方双击就能用。

### 想改工具 / 加业务图表

需要拿到**整个项目文件夹**（不止那个 HTML），然后参考下面的 [项目结构](#项目结构) 和 [添加业务图表](#添加业务图表)。改完源码后跑 `.\build.ps1` 重新生成 HTML。所有脚本都用相对定位，**项目放在哪个盘、哪个目录都能跑，不含任何绝对路径**。

## 功能

- **🆕 AI 生成图表** — 顶部「✦ AI 助手」打开对话框，用自然语言描述需求，AI 生成 Mermaid 代码并**自动填入代码区、即时渲染**；支持多轮对话连续修改、可选 4 种出图风格（详见 [AI 助手](#ai-助手用自然语言画图)）
- **🆕 图表样式** — 顶栏「🎨 样式」一键切换 **5 套配色**（暖橙 / 静谧蓝 / 森林绿 / 莫兰迪灰 / 糖果粉彩）、**连线**（圆润 / 直角 / 直线）、**字体**（无衬线 / 衬线 / 等宽）、**方向**（竖排 / 横排）；对手写、粘贴、AI 生成的图都即时生效（详见 [图表样式](#图表样式配色--连线--字体--方向)）
- **实时渲染** — 输入即出图，防抖 320 ms
- **13 种图表示例** — 流程图、序列图、类图、状态图、ER 图、甘特图、饼图、用户旅程、Git 图、思维导图、时间线、象限图，外加一个「样式演示」（classDef 上色 / 多种节点形状 / 分组）
- **画布交互** — 滚轮缩放（以光标为中心）、拖拽平移、双击或 ⊙ 适应窗口、`Ctrl + +/-/0` 快捷键
- **导出** — PNG（2 倍高清，带背景）、SVG、复制图片到剪贴板
- **友好的错误提示** — 识别行号 + 给出修复建议
- **自动保存草稿** — 关闭页面也不丢
- **深浅主题切换** — 暖米 / 暖深棕
- **沉浸阅读模式** — 左侧代码区可完全折叠 (`Ctrl + B`)，全屏看图

## 界面与基础操作

打开 `mermaid-studio.html` 后，界面分三块（从左到右同层并排）：

```
┌──────────────┬──────────────────┬───────────────┐
│  Code 代码区  │   Preview 预览区   │  AI 助手面板    │
│  写/粘 Mermaid │   实时渲染的图     │ （默认收起，    │
│              │                  │   点右上角打开） │
└──────────────┴──────────────────┴───────────────┘
        ↑ 中间分隔条可拖动调整宽度，AI 面板同理
```

**顶栏**（从左到右）：标题 · 「✦ AI 助手」按钮 · 载入封神图表 · 载入示例 · 折叠代码区 · 切换深浅主题 · 重置视图。

**代码区**：直接输入或粘贴 Mermaid 代码，右边即时出图（防抖 320ms）。顶部有「复制 / 清空」。`Tab` 键插入两个空格。出错时左侧行号会标红、预览区给出报错与修复建议。

**草稿**：内容自动存浏览器本地，关掉页面再开还在。

## AI 助手（用自然语言画图）

点顶部 **「✦ AI 助手」**（快捷键 `Ctrl + J`）在右侧**展开**对话面板——它和代码区、预览区**同层并排**，挤占空间而不是浮在上面遮挡内容，再按一次（或点面板右上角 `‹` 收起按钮）即可**收回**。面板和预览之间有一条可拖动的分隔条，能调整对话区宽度；窗口宽度、开合状态都会记住。用一句话描述你想要的图，AI 会生成 Mermaid 代码、自动填进左侧代码区并立即渲染。支持多轮对话——比如"把刚才那张图加一个支付失败的分支"，它会基于当前画布代码改。

> **重要前提**：AI 功能**不内置任何模型、不内置任何 Key**，也不替你联网。它用的是**你自己的大模型账号**——你得先有一个大模型的 API Key（云端或本地都行），在设置里填进去才能用。下面是详细的接入步骤。

### 一分钟上手

1. 双击打开 `mermaid-studio.html`
2. 点顶栏 **「✦ AI 助手」**，右侧展开对话面板
3. 点面板右上角 **⚙️**（或点底部"未配置模型 · 点击设置"）打开**模型设置**
4. 选一个**服务商**（会自动填好地址、模型、协议）→ 粘贴你的 **API Key** → 点**保存**
5. 在输入框里用大白话描述要画的图，回车发送。AI 生成代码 → 自动填入左侧 → 自动渲染

### 第一步：拿到大模型的 API Key

先去对应平台注册并创建 API Key（一串以 `sk-` 开头的密钥）。常见平台的 Key 获取入口：

| --- | --- | --- | --- |
| **DeepSeek** | <https://platform.deepseek.com> → 「API keys」 | `sk-...` | 便宜、中文好、对本工具友好 |
| **OpenAI** | <https://platform.openai.com/api-keys> | `sk-...` | 需要海外网络环境与付费余额 |
| **Claude** | <https://console.anthropic.com> → 「API Keys」 | `sk-ant-...` | 浏览器直连通常可用（见下文 CORS） |
| **通义千问（阿里百炼）** | <https://bailian.console.aliyun.com> → 「API-KEY」 | `sk-...` | 选「兼容 OpenAI」模式的地址 |
| **本地模型 Ollama** | 装好 [Ollama](https://ollama.com/) 后无需 Key | 随便填 | 完全离线、不花钱，见「本地模型」一节 |

### 第二步：在设置里填这 4 + 1 样东西

打开模型设置弹窗，从上到下：

| 字段 | 怎么填 |
| --- | --- |
| **服务商** | 选 DeepSeek / OpenAI / Claude / 通义千问 之一——选完会**自动帮你填好下面的地址、模型、协议**，通常你只需要再贴一个 Key。接其它平台选「自定义」。 |
| **API 地址 (Base URL)** | 服务的根地址。**OpenAI 兼容**服务填到 `/v1`（程序会自动追加 `/chat/completions`）；**Claude** 填到 `/v1`（自动追加 `/messages`）。别把 `/chat/completions` 自己加上去。 |
| **API Key** | 粘贴第一步拿到的密钥。右侧「显示」可临时查看是否贴错。**只存在你本机浏览器，绝不写文件、不上传、不进分享出去的 HTML。** |
| **模型名称** | 要调用的模型 ID，**必须和平台文档里的名字一字不差**（见下方速查表）。 |
| **接口协议** | OpenAI 兼容服务选 `/chat/completions`；Claude 选 `/messages`。**选预设服务商时会自动设好**，接「自定义」时按你的服务选。 |

填好点**保存**，面板底部的圆点会变绿、显示当前模型名，就代表配置就绪了。

### 本地模型（离线、免费、不卡跨域）

不想花钱或要完全离线，用本地模型最省心——本地服务一般不受跨域限制。

**Ollama**（推荐）：
1. 装 [Ollama](https://ollama.com/)，拉一个模型：`ollama pull qwen2.5`（或 `llama3.1` 等）
2. 让 Ollama 允许网页跨域访问：设置环境变量 **`OLLAMA_ORIGINS=*`** 后重启 Ollama
   （`file://` 页面的来源是 `null`，不放开会被拦）
3. 在本工具设置里：服务商选「自定义」、协议 OpenAI 兼容、地址填 `http://localhost:11434/v1`、模型填 `qwen2.5`、**Key 随便填**（如 `ollama`，但不能留空）

**LM Studio**：开启自带的本地服务器并**勾选 Enable CORS**，地址填 `http://localhost:1234/v1`，Key 填 `lm-studio`，模型填你加载的模型名。

> 提示：图表生成对模型有一定要求，**7B 以下的小模型容易写错语法**。本工具有自动修复兜底（见下），但更大的模型成功率更高。

### 团队共享一把 Key（可选 · 同事零配置）

如果团队想共用一把内部 Key，让同事打开就能用、不用每人各填，可以放一个**外置配置文件**——它不进仓库、不随 HTML 分享出去，避免误上传泄露：

1. 把仓库里的 **`ai_config.example.js`** 复制为同目录下的 **`ai_config.js`**
2. 打开 `ai_config.js`，把团队共享的 Key 填进 `apiKey`，按需改 `provider / baseURL / model / protocol`
3. 让 `ai_config.js` 和 `mermaid-studio.html` 放在同一目录，双击 HTML 即生效，**同事零配置直接用**

要点：

- **`ai_config.js` 已被 `.gitignore`（git）和 `svn:ignore`（svn）忽略**，不会进仓库、不会随分享出去的 `mermaid-studio.html` 外泄。仓库里只保留**不含 Key** 的模板 `ai_config.example.js`。
- **Key 绝不要写进 `template.html` / `mermaid-studio.html` / `.example`**——这几个文件会进仓库、会被分享，写进去就泄露了。
- 文件里的 Key 是**明文**，凡是拿到这个文件的人都能用、花费都算这把 Key，请只用**团队内部可共享**的 Key，别用个人付费私钥。
- **个人优先**：谁在 ⚙️ 设置弹窗里填过自己的配置，就以个人的为准，`ai_config.js` 只当**兜底默认**。
- **不想共享**？不创建 `ai_config.js` 即可，AI 助手会回退为「每人在设置里各填各的」，不影响使用（控制台会有一条无害的 404）。
- 改不改 `ai_config.js` **不需要重新 `build.ps1`**——它是独立外置文件，不参与构建。

### 第三步：好好提问（拿到更好的图）

发送框旁边有个**出图风格**下拉，决定 AI 这次怎么画（每次可单独选，会记住）：

- **简洁**：朴素结构，不加装饰
- **彩色强调**：用 `classDef` 给开始/结束/判断/异常等关键节点语义化上色
- **圆角柔和**：多用圆角 / 体育场形节点
- **分组蓝图**：用 `subgraph` 分组、区分主流程与支线

再加上顶栏的「🎨 配色方案」，同一张图能搭出很多种观感。提问技巧：

- **说清图的类型和要素**：「画一个**流程图**：用户下单 → 库存校验 → 支付 → 发货，支付失败要能回到下单」
- **指定方向/风格**：「用**从左到右**的流程图」「用**时序图**描述前端调用后端登录接口」
- **基于现有图改**：直接说「在刚才的图上加一个『退款』分支」——它会读当前画布代码改，不用重述
- **示例提示**（面板空白时也有可点的示例）：
  - 「画一个用户注册登录的流程图」
  - 「用时序图描述前端、后端、数据库三方完成一次登录的过程」
  - 「画电商订单状态机：待支付→已支付→已发货→已收货，含超时取消」

### 自动校验 + 自我修复（为什么基本不会再看到报错）

每次生成的代码会**先用 Mermaid 自带的解析器校验**，通过了才落到画布。万一模型输出了 PlantUML 之类的非 Mermaid 语法（`@startuml`、`start`、`:动作;`、`if-then-else-endif`、`note right:` 等，这些会导致 "No diagram type detected"），工具会**拦截并自动把报错回传给模型要求改正，最多自动修复 2 次**。只有能正常渲染的代码才会应用到画布；实在修不好就**保留你原来的图、不会用坏代码覆盖**，并在对话里提示你换种说法重试。

### 隐私与安全

- API Key 与对话记录**只保存在你本机浏览器的 localStorage**，不写入任何文件、不随 HTML 分享出去。
- 把 `mermaid-studio.html` 发给别人，**不会泄露你的 Key**（对方那份是空的，需各自配置）。
- 团队想共用 Key 的话，可放外置 `ai_config.js`（见上「团队共享一把 Key」）——它已被 `.gitignore` / `svn:ignore` 忽略，同样不进仓库、不随 HTML 外泄。
- 换机器、换浏览器、清缓存后需要**重新填配置**。
- 你的对话内容会直接发给你配置的那个大模型服务商，请遵守对应平台的数据政策。

### ⚠️ 跨域（CORS）问题与排查

本工具是 `file://` 双击运行的纯前端页面，浏览器直连云端 API 会受 **CORS（跨域）** 限制——**部分服务商不返回允许跨域的响应头，请求会被浏览器拦截**，表现为"调用失败 / Failed to fetch"。**这不是工具的 bug，是浏览器的安全策略。**

实测情况（可能随服务商策略变化）：

- **Claude（Anthropic）**：本工具已带 `anthropic-dangerous-direct-browser-access` 头，通常可直连
- **OpenAI**：一般可用
- **DeepSeek / 通义千问等**：取决于其是否放开 CORS，可能被拦
- **本地模型（Ollama / LM Studio）**：放开 CORS 后最稳（见上）

#### 常见报错对照表

| 现象 | 多半是什么原因 | 怎么办 |
| --- | --- | --- |
| 调用失败 / `Failed to fetch` / `NetworkError` | 跨域被拦，或网络不通 | 换支持跨域的服务商 / 用本地模型 / 走代理（见下）；也确认网络能访问该地址 |
| `HTTP 401` 未授权 | Key 填错、过期、或账户没余额 | 重新粘贴 Key、去平台确认 Key 有效且有额度 |
| `HTTP 404` | 地址或协议填错（少了 `/v1`、协议选反） | 核对「API 地址」和「接口协议」，对照速查表 |
| `HTTP 400` / model not found | 模型名拼错、或账号无该模型权限 | 改成平台文档里的准确模型名 |
| `HTTP 429` | 触发限流 / 欠费 | 稍后再试、或去平台充值提额 |
| 未在回复中找到 Mermaid 代码 | 模型没按格式输出 | 再发一次，或换更强的模型 |
| 生成的代码仍无法渲染（已自动修复 2 次） | 模型能力不足以写对语法 | 换更强的模型、把需求说得更简单具体 |

#### 跨域被拦时的几种绕过办法

1. **优先用本地模型**（Ollama / LM Studio，放开 CORS 后最稳）
2. 换一个**支持浏览器直连**的服务商
3. 用允许改响应头的浏览器扩展，或在本机起一个**反向代理**转发请求
4. 用 Chrome 的关闭 web security 调试模式启动（**仅自己测试用，别拿日常浏览器开**）

## 图表样式（配色 / 连线 / 字体 / 方向）

点顶栏 **「🎨 样式」** 打开样式面板，所有改动**即时预览**、会被记住，对手写 / 粘贴 / AI 生成的图都生效：

| 选项 | 可选值 | 说明 |
| --- | --- | --- |
| **配色方案** | 暖橙（默认）· 静谧蓝 · 森林绿 · 莫兰迪灰 · 糖果粉彩 | 一键给整张图换节点 / 连线 / 色块 / 饼图配色；每套都适配深浅两种界面主题 |
| **连线** | 圆润 · 直角 · 直线 | 流程图连线的弯曲方式（圆润=basis，直角=step，直线=linear） |
| **方向** | 竖排 ↓ · 横排 → | 流程图的排布方向（改写代码首行 `graph TD`/`graph LR`，仅对 flowchart 生效） |
| **字体** | 无衬线 · 衬线 · 等宽 | 图内文字字体 |

> 配色与右上角的「深浅主题」是两回事：深浅控制整个界面明暗，配色只换图表用色，两者可自由组合。

### Mermaid 还能玩哪些样式？（自己写代码时的建议）

如果你想手写出更好看的图，Mermaid v10.9.1 支持这些（点「载入示例 → 样式演示」可看一个综合例子）：

- **节点上色**：`classDef 名 fill:#7D9C68,stroke:#3D5A2E,color:#fff` 定义样式，再用 `A:::名` 或 `class A,B 名` 应用；也可单点 `style A fill:#bbf,stroke:#f66`
- **连线样式**：`linkStyle 0 stroke:#D97757,stroke-width:2px`（按连线出现顺序，从 0 计）
- **节点形状**：`[矩形]`、`(圆角)`、`([体育场])`、`[[子程序]]`、`[(数据库)]`、`((圆形))`、`{菱形判断}`、`{{六边形}}`、`[/平行四边形/]`
- **分组**：`subgraph 组名 ... end` 把相关节点框在一起
- **方向**：首行 `graph TD`（上下）/ `graph LR`（左右）/ `BT` / `RL`
- 让 **AI 助手** 帮你套用这些：在对话框选「彩色强调 / 圆角柔和 / 分组蓝图」风格即可（见下）

## 设计

"Anthropic warm paper" 风格：

| | |
| --- | --- |
| 主色 | 米色 `#F5F4ED` 纸面 |
| 文字 | 深棕 `#2A2620` |
| 强调 | Anthropic 橙 `#D97757` |
| 字体 | 衬线（Source Serif） / 无衬线（Inter） / 等宽（JetBrains Mono）混排 |

整套设计语言通过 CSS 变量在 `template.html` 顶部集中管理，调色只改 `:root.theme-warm` 和 `:root.theme-dim` 两个选择器中的变量。

## 项目结构

| 路径 | 角色 |
| --- | --- |
| `template.html` | 唯一前端源文件，所有 UI / 逻辑修改在这里做 |
| `mermaid.min.js` | 锁定版本 v10.9.1 的 mermaid 库（离线） |
| `fengshen-diagrams/*.mmd` | 业务图表，每个文件一张，构建时自动汇入下拉菜单 |
| `build.ps1` | 把上面三类源打包成 `mermaid-studio.html` 的构建脚本 |
| `mermaid-studio.html` | 最终产物（约 3.3 MB），双击即用 |

修改任何源（`template.html` 或 `fengshen-diagrams/*.mmd`）之后，回到项目根目录运行：

```powershell
.\build.ps1
```

`build.ps1` 用 `$PSScriptRoot` 自定位，**SVN / git checkout 到任何路径都能运行**，不依赖固定目录。

> **嫌手动跑烦？** 开发时可以在一个 PowerShell 窗口里开着 `.\watch.ps1`，它会监听 `template.html` 和 `fengshen-diagrams/*.mmd` 的保存事件，自动调用 `build.ps1`（400ms 防抖）。`Ctrl + C` 停止。

## 添加业务图表

下拉菜单"**载入封神图表**"的内容来自 [`fengshen-diagrams/`](./fengshen-diagrams) 文件夹。

```text
1. 在 fengshen-diagrams/ 新建一个 .mmd 文件     战斗循环.mmd
2. (可选) 首行写自定义显示名                    %% title: 战斗循环 · 主流程
3. 接下来写 Mermaid 代码                        graph TD ...
4. 在项目根运行 build.ps1                       .\build.ps1
5. 双击 mermaid-studio.html，下拉里已经有了
```

详见 [`fengshen-diagrams/README.md`](./fengshen-diagrams/README.md)。

> **为什么需要 build 步骤？** 浏览器在 `file://` 协议下不能列目录、也不能 fetch 兄弟文件（CORS 安全限制）。所以图表清单是构建时**静态嵌入**到 HTML 里的。这换来的好处是：最终 HTML 完全自包含，团队同事拿到 HTML 双击就能看到所有图表，不需要额外文件夹。
